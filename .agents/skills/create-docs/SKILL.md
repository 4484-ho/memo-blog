# create-docs（Docusaurus ドキュメント作成）

## いつ使うか

- `docs/` に新規ページを追加する依頼がある
- カテゴリ（サブディレクトリ）ごとにドキュメントを整理・追加する
- 表形式のサイト目次（`docs/index.md`）を最新に保つ必要がある

## 前提（このリポジトリ）

- サイドバーは [sidebars.ts](sidebars.ts) の `autogenerated` により **`docs/` のフォルダ構造から自動生成**される。通常、新規ファイル追加だけでよく、`sidebars.ts` は編集しない。
- [docusaurus.config.ts](docusaurus.config.ts) で `onBrokenLinks: 'throw'` のため、**目次・本文のリンクは必ず実在するパスに合わせる**。
- サイトの `baseUrl` は `/memo-blog/`。ドキュメントのルートは `/docs`。本文・目次の**相対リンク**はリポジトリ内の整合が取りやすい。

## 手順

### 1. カテゴリを決める

- トピックから論理カテゴリを 1 つ選ぶ（例: `cli`, `architecture`, `notes`）。
- **ディレクトリ名は `kebab-case`（英小文字とハイフン）を推奨**（URL にそのまま反映される）。
- 既に `docs/<category>/` がある場合はそれに合わせ、命名スタイルを既存に揃える。

### 2. ディレクトリを用意する

- `docs/<category>/` が存在しなければ作成する。

### 3. 新規 Markdown を作成する

- **パス**: `docs/<category>/<slug>.md`（`<slug>` も `kebab-case`）。
- **frontmatter**（必須に近い項目）:

```yaml
---
sidebar_position: 1
title: ページの表示タイトル
---
```

- **`sidebar_position`**: 同一 `docs/<category>/` 内で並び順を決める。既存ファイルがある場合は、そのカテゴリ内の最大の `sidebar_position` + 1。カテゴリ初回なら `1`。
- **`title`**: サイドバー等に出る**表示用タイトル**（日本語可）。**これを正とする。**
- **`description`**: SEO 用。必要なら任意で追加。
- **本文の見出し**: `title` と重複を避けるなら **H1（`#`）は書かない**か、`title` と同一文言の H1 を 1 つだけ置く。どちらかに統一する。

### 4. `docs/index.md` の表形式目次を更新する

- `## Index` 見出しの直下にある **Markdown テーブル**を維持・更新する。
- **列（固定）**

| 列名 | 内容 |
|------|------|
| カテゴリ | フォルダ名 `<category>`（表示はそのままか、必要なら短いラベル） |
| タイトル | 当該ドキュメントの frontmatter `title` と同じ文字列 |
| リンク | 当該 doc へのリンク（相対パス・**拡張子 `.md` は付けない**） |

- **リンク例**: `docs/cli/install.md` なら `[インストール](./cli/install)`。
- **新規ドキュメントを 1 つ追加したら、表に 1 行追加する**（重複行を作らない）。
- **並び順**: まずカテゴリ名（文字列ソート）、同一カテゴリ内は `sidebar_position` の小さい順。`sidebar_position` が同じならタイトル順。
- ルートの `docs/index.md` を表に載せる場合、相対リンクは Docusaurus の実 URL と一致させる（`./index` が `/docs/index` と解釈されビルドで壊れリンクになる構成がある）。**迷う場合は自己参照行を省略**し、`npm run build` で確認する。

### 5. 検証

- 追加したファイルパスと、目次テーブルのリンク先が一致しているか確認する。
- 可能ならリポジトリルートで `npm run build` を実行し、壊れたリンクがないか確認する。

## チェックリスト

- [ ] `docs/<category>/` が存在する（なければ作成した）
- [ ] `docs/<category>/<slug>.md` に `sidebar_position` と `title` がある
- [ ] `docs/index.md` の表に、新規 doc 用の行を追加（または初回のみ表のヘッダー整備）
- [ ] 目次リンクに `.md` を付けていない
- [ ] `npm run build` が通る（実行した場合）

## 新規ドキュメントのテンプレート例

```markdown
---
sidebar_position: 1
title: 例：セットアップ
---

## はじめに

（本文）
```

## やらないこと（この Skill の範囲外）

- `docusaurus.config.ts` の `editUrl` やデプロイ設定の変更
- `sidebars.ts` を手動カテゴリ定義に切り替える変更（特に依頼がない限り）
